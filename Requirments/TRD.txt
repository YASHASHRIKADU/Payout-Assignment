🏦 System: Playto Payout Engine (Minimal Production Design)
1. 🧩 System Overview

The Playto Payout Engine is a transaction-safe backend system that:

Maintains merchant balances using a ledger model
Processes payout requests with strong consistency guarantees
Ensures idempotent API behavior
Handles concurrent payout requests safely
Processes payouts asynchronously via a background worker

5. 💰 Ledger Computation

Balance is computed using database aggregation:

SELECT 
  COALESCE(SUM(CASE WHEN type='credit' THEN amount_paise END), 0)
  -
  COALESCE(SUM(CASE WHEN type='debit' THEN amount_paise END), 0)
AS balance
FROM ledger_entry
WHERE merchant_id = :merchant_id;
Key Guarantees:
No floating point errors
Always consistent with transaction history
Source of truth = ledger
6. 🔒 Concurrency Control
Problem:

Two concurrent payout requests may overdraft balance.

Solution: Row-Level Locking
from django.db import transaction

with transaction.atomic():
    merchant = Merchant.objects.select_for_update().get(id=merchant_id)

    balance = get_balance(merchant_id)

    if balance < amount:
        raise InsufficientBalanceException()

    payout = create_payout()
    create_ledger_debit()
Mechanism:
SELECT FOR UPDATE locks the merchant row
Ensures serialized access to balance check

8. 🔄 Payout State Machine
Allowed Transitions:
Current	Next
pending	processing
processing	completed
processing	failed
Enforcement:
def transition(payout, new_state):
    allowed = {
        "pending": ["processing"],
        "processing": ["completed", "failed"]
    }

    if new_state not in allowed.get(payout.status, []):
        raise Exception("Invalid state transition")

    payout.status = new_state
    payout.save()

    if result == "success":
